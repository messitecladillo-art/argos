# AI 多Agent Web 控制台 M1 测试分组与冒烟计划

## 1) M1测试目标
1. 验证 M1 核心链路可跑通：列表查看、详情查看、单播、群发、切换、恢复。
2. 验证多 agent 场景下状态、消息、结果归属正确，不串线。
3. 验证高频异常场景可见、可恢复，不影响主流程判断。
4. 在最小成本下识别发布阻断问题，形成冒烟优先级。

## 2) 测试分组
- 列表状态
- 详情时间线
- 单播发送
- 群发发送
- 切换恢复
- 异常处理

## 3) 每组检查点

### A. 列表状态
1. agent/profile 列表可正常加载。
2. 名称、状态、最近活跃时间展示完整。
3. 在线/忙碌/离线状态与后端一致。
4. 刷新后列表状态不异常、不丢失。
5. 空列表或加载失败时提示合理。

### B. 详情时间线
1. 进入单个 agent 后可看到最近聊天记录。
2. 输入/输出角色标识正确。
3. 时间线顺序正确，无重复无乱序。
4. 长文本、代码块、特殊字符展示正常。
5. 历史为空时空状态展示合理。

### C. 单播发送
1. 向单个 agent 发送指令成功发出。
2. 可看到发送中、处理中、成功/失败状态。
3. 结果只出现在目标 agent 会话中。
4. 重复点击不会造成非预期重复发送。
5. 失败时有明确提示与状态。

### D. 群发发送
1. 可选择多个 agent 并发送同一指令。
2. 各 agent 返回结果独立展示。
3. 支持部分成功/部分失败，不误报全部成功。
4. 离线 agent 失败不影响在线 agent 执行。
5. 群发结果可追溯到具体 agent。

### E. 切换恢复
1. 在多个 agent 间切换时，内容始终对应当前选中对象。
2. 快速切换下不串线、不残留前一个 agent 内容。
3. 页面刷新后，当前上下文可恢复。
4. 刷新后历史记录与发送结果不重复不丢失。
5. 重连后状态与详情重新加载正常。

### F. 异常处理
1. agent 离线时不可误显示发送成功。
2. 接口超时/500 时页面有明确错误提示。
3. 网络抖动或断网时页面不假死。
4. 空状态、异常状态、失败状态文案可理解。
5. 错误场景可在日志中定位。

## 4) 冒烟执行顺序
1. 列表状态：先确认列表能加载、状态能展示。
2. 详情时间线：进入 agent，确认会话可看、顺序正确。
3. 单播发送：验证最核心发送链路。
4. 群发发送：验证多 agent 协同链路与部分失败。
5. 切换恢复：验证切换、刷新、重连后的数据一致性。
6. 异常处理：最后验证离线、超时、报错、断网等高风险异常。

## 5) 阻断发布高风险项
1. 消息串线：消息或结果出现在错误 agent 会话中。
2. 越权访问：可看到未授权 profile/agent 的状态或记录。
3. 重复发送：一次操作触发多次非预期执行。
4. 状态误报：前端显示成功/在线，实际失败/离线。
5. 历史记录异常：刷新或重连后消息丢失、重复、乱序。
6. 群发结果错误：部分失败被显示为全部成功。
7. 异常不可恢复：超时、断网、500 后页面卡死或无法继续操作。
