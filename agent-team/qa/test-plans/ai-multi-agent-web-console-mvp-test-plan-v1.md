# AI 多Agent Web 控制台 MVP 测试计划 V1

## 背景
1. 本文档用于 AI 多Agent Web 控制台 MVP 阶段的测试准备与执行。
2. 当前 MVP 聚焦 4 项能力：查看多个 agent/profile 状态、查看聊天记录、agent 间切换、向单个或多个 agent 发送指令。
3. 文档目标是作为 `qa/test-plans/ai-multi-agent-web-console-mvp-test-plan-v1.md` 的正式引用稿。
4. 用于提测评审、执行对齐、回归控制和发布放行判断。

## 测试目标
1. 验证列表、会话、切换、单播、群发 5 条核心链路可用。
2. 验证多 agent / 多 profile 下数据展示正确，不串线、不误归属。
3. 验证异常场景可感知、可恢复，包括离线、超时、报错、部分成功。
4. 验证刷新、重连、快速切换下状态与历史记录保持一致。
5. 拦截越权、重复发送、状态误报、记录异常等上线阻断问题。

## 测试范围
1. 多个 Hermes agent/profile 的列表展示与状态刷新。
2. 单个 agent 输入输出与聊天记录查看。
3. agent 间切换与当前上下文一致性。
4. 向单个 agent 发送指令及状态流转。
5. 向多个 agent 群发指令及逐 agent 结果展示。
6. 页面刷新、重连后的状态与历史记录恢复。
7. 常见异常处理：离线、超时、500、空状态、重复提交、长文本。
8. 基础权限隔离、错误日志、执行日志可追踪性。

## 不测范围
1. 非 MVP 功能，如复杂任务编排、细粒度运营能力。
2. agent 内部模型效果与内容质量评估。
3. 大规模性能压测，仅做基础并发验证。
4. 移动端与小屏专项适配，如当前仅面向桌面 Web。
5. 极老旧浏览器兼容性，如无明确上线要求。
6. 第三方服务本身正确性，仅验证接入后的控制台表现。

## 测试环境
1. 准备测试环境、预发环境各 1 套，版本可追踪。
2. 准备在线、忙碌、离线 3 类 agent，覆盖至少 2 个 profile。
3. 准备正常权限账号、受限权限账号各 1 类。
4. 准备历史消息、长文本、特殊字符、失败样本等固定测试数据。
5. 支持注入超时、500、断网、agent 不可达、群发部分失败。
6. 前端错误日志、后端接口日志、agent 执行日志需可查。
7. Chrome 最新版必测，Safari/Edge 视要求补测。

## 测试策略
1. 先冒烟，再主流程，再异常流，最后回归与放行检查。
2. 以用户主路径为主：列表 → 会话 → 切换 → 单播 → 群发 → 刷新恢复。
3. 以高风险问题优先：串线、越权、重复发送、状态误报、记录异常。
4. 用例按模块组织：列表、会话、切换、单播、群发、恢复、异常、权限。
5. 每条用例包含前置条件、步骤、预期结果、优先级、是否回归必跑。
6. 冒烟覆盖核心主流程；回归覆盖核心主流程 + 高风险异常。
7. 结果校验采用 UI 展示 + 接口/日志双重确认。

## 冒烟
1. 控制台成功加载 agent/profile 列表。
2. 成功进入任一 agent 并查看最近聊天记录。
3. 在多个 agent 间切换，内容始终对应当前对象。
4. 向单个 agent 发送指令并看到结果反馈。
5. 向多个 agent 群发指令并看到各自独立结果。
6. 刷新页面后，状态、上下文、历史记录可恢复。
7. 离线或接口失败时，页面有明确提示且不假死。

## 回归
1. 列表展示、状态刷新、最近活跃时间正确。
2. 聊天记录顺序、角色、时间戳、长文本展示正确。
3. 单播发送稳定，重复点击不会造成重复执行。
4. 群发发送稳定，支持部分成功，结果归属正确。
5. 快速切换下内容不混杂、不串线。
6. 刷新/重连后消息不丢失、不重复、不乱序。
7. 异常提示明确：离线、超时、500、空状态、断网。
8. 受限账号不能查看未授权 profile/agent 数据。

## 缺陷准入与发布准出
1. 提测前需保证列表、会话、切换、单播、群发链路可执行。
2. 提测前需保证环境稳定、日志可查、账号和样本数据已就绪。
3. 如群发、权限、恢复链路未打通，不进入正式提测。
4. 提测版本需提供版本号、构建时间、变更说明、已知问题。
5. 发布准出要求 P0/P1 缺陷为 0。
6. P2 缺陷需有影响评估、规避方案、责任确认。
7. 冒烟全通过，核心回归通过，上线阻断项全部关闭。
8. 日志、监控、告警、回滚预案需提前确认。

## 风险与观测项
1. 风险：消息串线。观测：核对 UI、接口、日志中的 agent/profile/消息 ID。
2. 风险：状态误报。观测：对比前端状态、接口结果、agent 实际执行日志。
3. 风险：重复发送。观测：检查重复点击、重试、断网恢复后的任务数量。
4. 风险：历史记录异常。观测：检查刷新、重连、切换后的消息总数与顺序。
5. 风险：群发部分失败被掩盖。观测：核对聚合态与逐 agent 明细态。
6. 风险：权限隔离不足。观测：用受限账号检查可见范围与接口返回。
7. 风险：长时间运行不稳定。观测：观察卡顿、状态停更、日志异常。
8. 风险：异常处理不完整。观测：模拟超时、500、断网、离线后检查恢复能力。

## 附录建议
1. 测试环境清单：环境地址、版本号、账号、agent/profile 样本。
2. 测试数据清单：历史会话、长文本、特殊字符、失败样本、权限样本。
3. 用例编号规范：MVP-LIST / CHAT / SWITCH / SEND / BROADCAST / RECOVER / AUTH。
4. 缺陷分级标准：P0、P1、P2、P3 定义统一。
5. 发布检查记录：版本号、冒烟结果、回归结果、遗留缺陷、放行结论。
