# AI 多Agent Web 控制台 M1 验收清单

## 1) M1验收目标
1. 能在首页看到多个 agent 的列表和当前状态。
2. 能进入单个 agent 详情查看独立时间线。
3. 能向单个 agent 发送指令并看到结果或失败提示。
4. 能在不同 agent 间切换且不串消息。
5. 能通过群发入口对多个 agent 分别发送同一指令。
6. 遇到空态、加载态、错误态时页面有明确反馈。

## 2) 页面与模块拆分
### 首页 / agent列表
1. 顶部全局栏
2. agent 列表区
3. 状态标签区
4. 最近活动摘要
5. 刷新入口

### 详情页 / 时间线
1. 顶部 agent 信息栏
2. 聊天/事件时间线
3. 历史加载区
4. agent 快速切换入口

### 发送区
1. 文本输入框
2. 发送按钮
3. 发送中状态
4. 发送失败提示/重试

### 群发入口
1. 多选 agent 区
2. 统一输入区
3. 发送确认区
4. 逐项结果区

### 异常态
1. 加载态
2. 空态
3. 接口失败态
4. agent offline/error 态
5. 重试入口

## 3) 每个模块验收点
### 首页 / agent列表
1. 能展示 agent 名称、角色、状态、最近活动时间。
2. 状态变化后可刷新看到最新结果。
3. 点击任一 agent 能进入对应详情页。

### 详情页 / 时间线
1. 只展示当前 agent 的消息和事件。
2. 消息类型可区分 input/output/system/error。
3. 加载更多后历史记录追加正常，不重复不串页。

### 发送区
1. 输入指令后可发送到当前 agent。
2. 发送后消息立即进入时间线。
3. 返回结果或失败提示可见。
4. 发送失败时可重试。

### 群发入口
1. 可选择多个 agent。
2. 同一指令会分别发送到每个目标 agent。
3. 每个 agent 都有独立发送结果。
4. 部分失败时能看到失败项。

### 异常态
1. 无数据时有空态文案。
2. 加载中时有 loading 反馈。
3. 接口失败时有错误提示和重试入口。
4. agent offline/error 时列表和详情都可见。

## 4) P0/P1 标注
### P0
1. 首页 agent 列表与状态展示。
2. 详情页时间线展示。
3. 单 agent 发送区。
4. agent 间切换。
5. 基础异常态：loading/empty/error/offline。
6. 刷新机制。

### P1
1. 群发发送入口。
2. 最近活动摘要优化。
3. 历史加载体验优化。
4. 逐项发送结果展示优化。
5. 重试与失败反馈细化。

## 5) 给 engineer 的实现边界提醒
1. M1 先解决“能看、能切、能发、能报错”，不要先做复杂编排。
2. 群发是“分别发送”，不是任务编排或结果聚合分析。
3. 所有消息必须按 agent 隔离，不能共享时间线状态。
4. 先用轮询/手动刷新，不强依赖实时流式方案。
5. 先做桌面 Web 主流程，不扩展移动端和权限体系。
6. UI 必须先覆盖主态、空态、加载态、错误态，再做美化。
