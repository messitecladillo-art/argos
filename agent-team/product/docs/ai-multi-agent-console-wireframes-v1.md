# AI 多Agent Web 控制台页面线框说明 v1

## 文档目标
- 用于设计与前端执行准备。
- 覆盖首页、详情页、群发入口、异常态与信息层级。
- 默认面向桌面 Web MVP。

## 控制台首页 / Agent 列表页线框说明
1. 顶部全局栏：展示页面标题、全局状态、刷新按钮、群发发送入口。
2. 主体为 agent 列表区：按行或卡片展示各 agent。
3. 每个 agent 显示：名称、profile/角色、状态、最近活动时间、最近消息摘要。
4. 点击任一 agent，可进入对应详情页。
5. 可保留最近访问 agent 的快捷入口。
6. 无 agent 时展示空态提示和刷新入口。

## Agent 详情 + 聊天时间线页线框说明
1. 顶部信息栏：展示返回入口、agent 名称、profile/角色、当前状态、最近更新时间。
2. 中间主区域为聊天/事件时间线：展示 input、output、system、error。
3. 时间线按时间顺序展示，并区分消息来源和类型。
4. 页面支持加载更多历史记录。
5. 页面内提供其他 agent 的快速切换入口。
6. 底部固定输入区：包含输入框、发送按钮、发送中/失败反馈。

## 群发发送入口线框说明
1. 顶部说明区：说明该入口会把同一条指令分别发送给多个 agent。
2. agent 选择区：支持多选目标 agent，并展示名称和状态。
3. 指令输入区：提供统一文本输入框。
4. 发送确认区：展示已选数量和发送按钮。
5. 结果反馈区：逐个展示每个 agent 的发送结果。
6. 可提供二次确认，降低误发风险。

## 异常空态加载态说明
1. 加载态：首页列表、详情时间线、群发发送过程都要有明确 loading 反馈。
2. 空态：无 agent、无消息、未选择群发对象时，都要有说明文案。
3. 异常态：agent offline、agent error、接口加载失败、消息发送失败都要单独提示。
4. 异常态需提供重试入口，如刷新、重新发送、重新加载。
5. 群发失败时需逐项展示失败对象，不能只给总提示。
6. 所有异常/空态都不能出现无反馈空白页。

## 区域优先级与信息层级
1. 第一优先级是当前是否可操作：状态、错误、发送是否可用。
2. 第二优先级是当前对象是谁：当前 agent、目标 agent、当前页面上下文。
3. 第三优先级是最近发生了什么：最近消息、最近活动、最近错误事件。
4. 第四优先级是辅助信息：profile 描述、最近访问、扩展信息。
5. 首页优先看 agent 列表和状态，再看快捷入口和摘要信息。
6. 详情页优先看时间线和输入区，再看辅助信息与跨 agent 切换。
